DIY Multimeter

Mar 8, 2021 - Apr 19, 2021

This project was actually a lab for a course called Engineering Analysis and Practise (ELET 3405). We were given a schematic for a multimeter using an ATmega 32 or ATmega 16 at the core and tasked with implementing the firmware. The specific features each student had to implement were determined by their major. One of my majors was in Electronics hence I had to implement firmware to measure temperature, frequency of a square wave and distance in addition to the default DC voltage, capacitance and resistance. Unfortunately due to the course being delivered remotely, a lack of access to the necessary modules and the lecturer having external issues hindering his ability to moderate the course, the deadline was extended beyond the typical week long duration. Single GPU Passthrough

Oct 2021

At this point I had been dual booting Windows and Linux for almost 3 years and realized the only real need I had for Windows was for proprietary software with no FOSS equivalent offered in my distro's repository and for the rare occasions that I play games. Realizing this, the idea of switching to linux full time and using a virtual machine with hardware passthrough for Windows became more appealing versus having to maintain 2 OSes, one of which I rarely use. This project served as a final experiment to explore the feasibility of running Windows in a disposable VM versus dual booting. After spending a few days learning how to passthrough hardware with my hypervisor and actively using it for my specific use cases I found the advantages far outweighed the disadvantages for my specific situation and I have since switched to linux full time in the following month.
Advantages Disadvantages
saves storage space as the virtual disk expands as needed less system resources (like RAM and CPU cores) available to the VM
not limited to storing personal files on NTFS (which has limited support for linux at the time of writing this) or FAT file systems. In other words I can use more complex file systems with more features like BTRFS with copy-on-write and snapshot support. exposing files to VM via a network share over a virtual network has noticeably higher latency than passing through a drive
I can use full disk encryption to protect my personal files when my PC is off the performance penalty for virtualization (though this is mainly noticeable in more demanding tasks)
less drivers to install and update on a regular basis
easy to backup and restore a VM by using snapshots or saving a copy of the virtual disk and the hypervisor configuration
For details on how I setup the VM see this repository.

MAX-30102 and MPU-6050 Arduino Libraries

Mar 2020 - Apr 2020

These libraries were developed to facilitate another project I was working on simultaneously intended to monitor patient's well-being. The patient monitoring system tracked the patient's heart rate, sleeping position and detect falls among other things and relay the data to a server. While I can't release the full code for the project I thought at least publish the libraries for anyone who may need them in the future. Libraries had already existed for both modules however the communication via i2c and the logic to interface with the modules were deeply coupled preventing both libraries from being used simultaneously. Personally, I also found the code not to be very readable so I thought to rewrite the libraries as opposed to modifying the existing code. It didn't take long to implement a functional interface (I finished both in about a week) and most of the time was spent researching algorithms to detect heartbeats from the raw IR readings from the MAX 30102 followed by orientation and fall detection from the readings from the MPU-6050.
